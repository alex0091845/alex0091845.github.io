Chapter 1.4. Writing Methods and Using "this"

Short summary: this refers to the calling object.

To tie the whole "individual" concept together, understanding the `this` keyword is important.

You might have seen some code that has the this keyword in it.

Code with this keyword.

What is it referring to? Who's this? (Oh no, not another knock knock joke)

Let's look at a couple of examples:

Example 1

When the run method is called on tommy (that is, tommy.run()), we're basically telling tommy to follow the instructions as written in the method run.

So when Tommy reads the instructions written in the run method: "Quickly take one big step with your right leg."

Tommy thinks: "your?" Who's the "you" that they're talking about?

Then he goes, Oh wait, I am the "you" that they're referring to!

So he takes one big step with his right leg.

Obviously, there's also the left leg, and back to right, and so on. But we're not interested in that in particular right now.

Example 2

Now let's say we also tell isabelle to run, aka isabelle.run().

When Isabelle reads the run instructions, again it says "Quickly take one big step with your right leg."

Isabelle thinks: "your?" Who's the "you" that they're talking about?

Then she goes, Oh wait, I am the "you" that they're referring to!

So she takes one big step with her right leg.

...and so on.

So who's the "you" referring to? We don't know who will read it, but it refers to whoever is reading the instructions we wrote!

When Tommy reads and follows the instructions, he knows the "you" is referring to him.

When Isabelle reads and follows the instructions, she knows the "you" is referring to her.

Just as in the real world, you know who I'm referring to when I say "you."

Section Summary

In the method definition, this refers to the calling object (whatever is in front of the dot operator (.))

When the CPU runs the line tommy.run(), tommy is the calling object to the run method. So inside of the run method definition, this refers to tommy.

Similarly, when the CPU runs the line isabelle.run(), isabelle is the calling object to the run method now. So inside of the run method definition, this refers to isabelle.

So in summary, this is like "you", and refers to whoever is taking the action (calling the method).

IMPORTANT: we don't know who this refers to until the method is called (aka someone reads it/takes the action/calls the method!)
